The Security Server is the single gateway between an information system and the network, and it encapsulates the security mechanisms below. Configure and verify them; never bypass them.

  • PKI (X.509). Every Security Server holds authentication certificates for the inter-server channel and signing certificates for member identity on each message.
  • Mutual TLS (mTLS). Security Servers authenticate each other with mutual certificate-based TLS before any message flows. The network is never trusted.
  • Message signing. Every message is cryptographically signed as digital evidence. Unsigned or tampered messages are rejected.
  • OCSP validation. Certificate validity is checked through OCSP. Revoked or expired certificates fail closed.
  • Timestamp validation (TSA, RFC 3161). Messages and signatures are logged and batch-timestamped for long-term proof.
  • Signed global configuration. The Central Server signs the global configuration; Security Servers reject unsigned or modified configuration.
OperatorRuns the X-Road instance services: Central Server governance, management service provider, trust setup, approvals and monitoring. In this sandbox cs runs the Central Server and ss-mtc hosts MANAGEMENT.
ConsumerCalls a service through its own Security Server. It sends the X-Road-Client identity and must have an ACL grant before the provider accepts the request.
ProviderPublishes a service behind its Security Server. It owns the real information system/database and decides which consumers get access.
